Student Authentication API

Welcome to the Student Authentication REST API. This API provides endpoints for user authentication and CRUD operations on student data.

API Endpoints

Authentication

POST /api/auth/register - Register a new user
Request Body:
{ "nama": "Budi", "nim": "123", "password": "123456" }
POST /api/auth/login - Login user
Request Body:
{ "nim": "123", "password": "123456" }

User Management (CRUD)

GET /api/users - Get all users
GET /api/users/:id - Get user by ID
POST /api/users - Create new user
Request Body:
{ "nama": "Andi", "nim": "999", "password": "123" }
PUT /api/users/:id - Update user
Request Body:
{ "nama": "Andi Update", "password": "456" }
DELETE /api/users/:id - Delete user

Testing with REST Client

For testing API endpoints, you can use the .http files in the rest-client/ folder:

  • rest-client/auth.http - Authentication tests
  • rest-client/users.http - User CRUD tests

Features

  • Password hashing using bcrypt
  • Input validation
  • Unique NIM constraint
  • Consistent JSON response format
  • REST Client integration for testing
